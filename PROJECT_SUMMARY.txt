# PROJECT SUMMARY: SV Satellite Platform

## 🌟 Overview
The SV Satellite Platform is a professional-grade, dual-app ecosystem designed to streamline the operations of an Internet Service Provider (ISP) or Satellite TV provider. It bridges the gap between customer service requests and administrative management with a modern, cloud-based architecture.

## 🛠 Technology Stack
This project utilizes a high-performance, modern technology stack for maximum reliability and scalability:

### **Programming Languages**
- **JavaScript (ES6+)**: Used for the entire core application logic, state management, and interaction flows.
- **JSON**: Used for metadata, app configurations, and project manifests.
- **Markdown**: Used for comprehensive technical and project documentation.

### **Core Resources & Frameworks**
- **Expo & React Native**: Powering both the Customer and Admin apps as cross-platform mobile applications.
- **Google Firebase**:
    - **Authentication**: Provides secure, multi-method login (Phone/Email).
    - **Cloud Firestore**: A real-time NoSQL database ensuring data consistency across all devices.
- **OneSignal**: Enterprise-level push notification service for real-time alerts and user engagement.
- **EAS (Expo Application Services)**: Utilized for automated cloud-based building, signing, and Play Store submission pipelines.

## 📱 Application Modules

### 1. Customer Application (DishCustomerApp)
*Goal: Empowerment through self-service.*
- **Registration**: Streamlined onboarding with mobile verification and location profiling.
- **Subscription Tracking**: Real-time view of active plans, speeds, and "Days Remaining."
- **Plans Discovery**: Exploration of tiered service packages and add-ons.
- **Support Tickets**: Integrated issue reporting with image attachment capabilities.
- **Multi-language Support**: Fully localized to serve a diverse user base.

### 2. Administrator Application (DishAdminApp)
*Goal: Total operational control.*
- **User Approval**: Centralized queue to review and authorize new customer registrations.
- **Billing Management**: Specialized **BCN Calculator** logic for prorated renewals and fixed-fee billing.
- **Broadcast System**: Ability to send mass notifications to all users for maintenance or updates.
- **Real-time Monitoring**: Instant alerts for new payments, user registrations, and support tickets.

## ⚙️ Business Logic & Flow
1. **Onboarding**: A customer registers; the system places them in a "Pending Verification" state.
2. **Authorization**: Admins receive a OneSignal notification and approve the profile after verifying details.
3. **Service Lifecycle**: Customers manage their subscriptions, while Admins handle the billing renewals and support fulfillment.
4. **Data Integrity**: Global security rules ensure data privacy between customers while giving owners a bird’s-eye view of the organization.

## 🏆 Project Highlights
- **Consolidated Monorepo**: Both platforms live in a single repository for unified maintenance.
- **Automated CI/CD**: Production-ready pipelines for immediate Play Store deployment.
- **Resilient Architecture**: Built-in specialized utilities for image optimization, offline caching, and database maintenance.
